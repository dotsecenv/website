This guide takes you from zero to an encrypted, auto-loading secret in about 5 minutes.

By the end you will have dotsecenv installed, a vault with your identity, a stored secret, and a `.secenv` file that loads it automatically when you enter your project directory.

## Prerequisites

You need a GPG key. If you do not have one, run `dotsecenv identity create` after step 3 below.

## Steps

<Steps>

1. **Install dotsecenv**

   ```bash
   curl -fsSL https://get.dotsecenv.com/install.sh | bash
   ```

   The installer downloads the binary, verifies signatures, and installs the shell plugin. At the end it prints plugin setup instructions.

2. **Activate the shell plugin**

   Add the source line the installer printed to your shell configuration:

   <Tabs>
     <TabItem label="Zsh">
       ```bash
       echo 'source ~/.local/share/dotsecenv/plugin/dotsecenv.plugin.zsh' >> ~/.zshrc
       ```
     </TabItem>
     <TabItem label="Bash">
       ```bash
       echo 'source ~/.local/share/dotsecenv/plugin/dotsecenv.plugin.bash' >> ~/.bashrc
       ```
     </TabItem>
     <TabItem label="Fish">
       The installer links the plugin automatically for Fish. If it did not, run:

       ```bash
       ln -sf ~/.local/share/dotsecenv/plugin/conf.d/dotsecenv.fish ~/.config/fish/conf.d/dotsecenv.fish
       ```
     </TabItem>
   </Tabs>

   Reload your shell:

   ```bash
   exec $SHELL
   ```

   <Aside type="tip">
   If you use Oh My Zsh, Zinit, Fisher, or Oh My Bash, the installer auto-links the plugin. Just add `dotsecenv` to your plugins list in your rc file.
   </Aside>

3. **Initialize configuration, vault, and login**

   ```bash
   dotsecenv init config
   # Output: Created /home/you/.config/dotsecenv/config

   dotsecenv init vault
   # Output: Created /home/you/.local/share/dotsecenv/vault

   dotsecenv login
   # Prompts you to select a GPG key, then:
   # Output: Logged in as Your Name <you@example.com>
   ```

4. **Store a secret**

   ```bash
   echo "s3cr3t-db-pass" | dotsecenv secret store DATABASE_PASSWORD
   ```

5. **Verify it works**

   ```bash
   dotsecenv secret get DATABASE_PASSWORD
   # Output: s3cr3t-db-pass
   ```

6. **Create a project with a `.secenv` file**

   ```bash
   mkdir -p ~/my-project
   echo 'DATABASE_PASSWORD={dotsecenv}' > ~/my-project/.secenv
   ```

   This tells the shell plugin to fetch `DATABASE_PASSWORD` from your vault whenever you enter the directory.

7. **Watch it auto-load**

   ```bash
   cd ~/my-project
   # dotsecenv: found .secenv in /home/you/my-project
   # Load secrets? [y]es / [n]o / [a]lways: a
   # dotsecenv: loaded 1 secret(s) from .secenv: DATABASE_PASSWORD

   echo $DATABASE_PASSWORD
   # Output: s3cr3t-db-pass

   cd ~
   # dotsecenv: unloaded 1 secret(s): DATABASE_PASSWORD

   echo $DATABASE_PASSWORD
   # Output: (empty)
   ```

</Steps>

## What just happened

- Your secret is **encrypted at rest** in the vault using your GPG key
- The shell plugin **auto-loads** secrets when you enter a directory with `.secenv`
- Secrets are **auto-unloaded** when you leave the directory tree
